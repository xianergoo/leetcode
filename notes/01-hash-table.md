# 哈希表 (Hash Table) 专题笔记

## 1. 核心概念

哈希表是一种 **键值对 (key-value)** 数据结构，通过哈希函数将 key 映射到存储位置，实现 O(1) 的平均查找/插入/删除。

```
┌──────────────────────────────────────┐
│  key  ──→  哈希函数  ──→  索引位置   │
│  "abc"  →  hash("abc") →  bucket[3] │
└──────────────────────────────────────┘
```

## 2. Go 中的 map 用法

```go
// 创建
m := make(map[int]int)
m := make(map[string]int, 16)  // 预分配容量

// 增删改查
m[1] = 100                  // 增/改
delete(m, 1)                // 删
val, ok := m[1]             // 查（ok 表示是否存在）
if _, ok := m[1]; !ok { }   // 判断是否存在

// 遍历
for k, v := range m { ... }
```

**关键点：**
- map 是**无序**的，遍历顺序不固定
- 访问不存在的 key 返回零值，不会 panic
- 不是并发安全的

## 3. 哈希表的三大解题模式

### 模式 1：两数之和型 — 「存值找互补」
> 遍历数组，把当前元素存入 map，同时查找 target - nums[i] 是否存在

**题例：** 1. 两数之和

```go
// 核心：一次遍历，边存边查
m := make(map[int]int)
for i, v := range nums {
    if j, ok := m[target - v]; ok {
        return []int{j, i}
    }
    m[v] = i
}
```

### 模式 2：频次统计型 — 「统计 + 判断」
> 统计每个元素出现的次数，然后基于频次做判断

**题例：** 49. 字母异位词分组, 169. 多数元素

```go
// 统计出现次数
count := make(map[int]int)
for _, v := range nums {
    count[v]++
}
// 然后按频次筛选或判断
if count[v] > len(nums)/2 { ... }
```

### 模式 3：集合去重型 — 「存在性判断」
> 用 map 当 set 用（Go 没有内置 set），快速判断元素是否存在

**题例：** 128. 最长连续序列

```go
set := make(map[int]bool)
for _, v := range nums {
    set[v] = true
}
if set[x-1] { ... }  // O(1) 判断是否存在
```

### 模式 4：前缀和 + 哈希 — 「区间和统计」
> 用 map 记录前缀和及其出现次数，统计满足条件的子数组个数

**题例：** 560. 和为 K 的子数组

```go
preSum := make(map[int]int)
preSum[0] = 1  // 空数组前缀和为 0
sum, count := 0, 0
for _, v := range nums {
    sum += v
    count += preSum[sum - k]  // 找互补前缀和
    preSum[sum]++
}
```

## 4. 复杂度速查

| 操作 | 平均 | 最坏 |
|------|------|------|
| 查找 | O(1) | O(n) |
| 插入 | O(1) | O(n) |
| 删除 | O(1) | O(n) |

最坏情况发生在哈希冲突严重时（Go 的 map 有较好的冲突处理，实际很少出现 O(n)）。

## 6. 易错点

1. **map 取不存在的 key** — 返回零值而非报错，可能误判
   → 始终用 `val, ok := m[key]` 模式
2. **遍历时修改 map** — 可能 panic 或产生不确定行为
3. **key 类型限制** — slice/map/function 不能作为 key（不可比较类型）
